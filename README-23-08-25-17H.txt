## 🎯 Vue d'ensemble

**JustiJob** est une plateforme révolutionnaire qui utilise l'IA pour démocratiser l'accès à la justice du travail. Notre système permet aux employés de :
- ✅ Évaluer leurs chances de succès aux Prud'hommes
- ✅ Obtenir un dossier juridique complet de 30 pages
- ✅ Bénéficier d'une analyse IA basée sur la jurisprudence
- ✅ Économiser 90% des frais d'avocat

### Chiffres clés
- **85%** de taux de succès
- **2 minutes** pour le diagnostic
- **90€** pour un dossier complet (vs 1000€+ chez un avocat)
- **24h** de délai de traitement

## 💻 Installation

### Prérequis
- Node.js 18.0 ou supérieur
- npm ou yarn
- Windows, macOS ou Linux

### Étapes d'installation

```bash
# 1. Cloner le projet (si applicable)
git clone [votre-repo]
cd justijob-nextjs-version-3

# 2. Installer les dépendances
npm install

# 3. Installer react-dropzone (si pas déjà fait)
npm install react-dropzone

# 4. Créer le fichier environnement (optionnel)
# Créez .env.local avec :
# STRIPE_PUBLIC_KEY=pk_test_xxxxx
# STRIPE_SECRET_KEY=sk_test_xxxxx

# 5. Lancer le serveur de développement
npm run dev

# 6. Ouvrir dans le navigateur
# http://localhost:3000
```

## 🔄 Workflow complet

### Parcours utilisateur type

```mermaid
graph LR
    A[Accueil] --> B[Diagnostic Gratuit]
    B --> C[Score & Résultats]
    C --> D{Décision}
    D -->|Intéressé| E[Paiement 90€/45€]
    E --> F[Téléchargement Questionnaire]
    F --> G[Remplissage Offline]
    G --> H[Upload Documents]
    H --> I[Analyse IA]
    I --> J[Réception Dossier 24h]
    D -->|Pas intéressé| K[Fin]
```

### Documents requis
1. **Questionnaire JustiJob rempli** (obligatoire)
2. **Contrat de travail** (obligatoire)
3. **Dernières fiches de paie** (obligatoire)
4. **Documents supplémentaires** (optionnel)

## 🔌 APIs disponibles

### 1. `/api/diagnostic`
- **Méthode** : POST
- **Fonction** : Calcule le score basé sur les réponses
- **Retour** : Score, niveau, points forts/faibles

### 2. `/api/upload`
- **Méthode** : POST
- **Fonction** : Upload sécurisé des documents
- **Retour** : ID de référence pour le suivi

### 3. `/api/payment`
- **Méthode** : POST
- **Fonction** : Traitement du paiement (simulation)
- **Retour** : Confirmation et ID de transaction

### 4. `/api/generate-dossier`
- **Méthode** : POST
- **Fonction** : Génération du dossier complet
- **Retour** : Dossier JSON avec estimations
